Finance Tracker Project Overview

A summary of the project's architecture, features, and key code implementations.

Core Functionality

The application is designed to help users track their personal finances by managing income and expenses through a system of transactions and categories.

  • User Authentication (Registration & Login)
  • Full CRUD (Create, Read, Update, Delete) operations for Transactions.
  • Full CRUD operations for Categories, with special protections for the default category.
  • Dynamic server-side sorting for data lists.

Technology Stack

Backend

  • C# & .NET
  • ASP.NET Core Web API
  • Entity Framework Core (for data access)
  • PostgreSQL (Database)
  • AutoMapper (for DTO mapping)

Frontend

  • Angular & TypeScript
  • Angular Material (for UI components like MatSnackBar)
  • SCSS for styling

DevOps & Deployment

  • Docker (for application containerization)
  • Azure App Service (for cloud deployment)

Code Implementation Highlights

Data Integrity: Transaction & Category Types

Implemented critical business logic on the backend to maintain data consistency. The system enforces that 'Income' transactions can only exist in 'Income' or 'Neutral' categories, and 'Expense' transactions in 'Expense' or 'Neutral' categories.

If a user action violates this rule (e.g., changing a category's type), the backend automatically re-assigns the affected transactions to the user's "WITHOUT CATEGORY" default category. This logic is handled in the CategoryController and TransactionController.

Efficient DTO Mapping with AutoMapper

To streamline the process of converting data models (Entities) to Data Transfer Objects (DTOs) for API responses, the project leverages AutoMapper. This library automates the mapping of object properties, significantly reducing repetitive boilerplate code in the controllers and making the codebase cleaner and more maintainable.

Containerized Deployment with Docker & Azure

The entire application is containerized using Docker. This encapsulates the application and its dependencies, ensuring a consistent environment from local development to production. These Docker images are deployed to Azure App Service, a fully managed platform that simplifies deployment, scaling, and monitoring, providing a robust and reliable hosting solution in the cloud.

Centralized Constants

To avoid hardcoding magic strings, a static Constants class was created in the backend project (/Utils/Constants.cs). This holds shared values like the default category name, making the code cleaner and easier to maintain.

Dynamic Server-Side Sorting

The API endpoints for fetching categories and transactions were enhanced to accept sortBy and sortOrder query parameters. The backend now performs the sorting using LINQ, ensuring the frontend receives data in the desired order efficiently. This reduces the processing load on the client.

UI/UX & User Confirmations

On the frontend, user experience is improved by providing clear warnings for actions with side effects. When a user changes a category's type, an MatSnackBar appears with a "Confirm" action, warning them that some transactions may be moved. This prevents accidental data changes. Custom styles were applied to the snackbar for a consistent look and feel.
